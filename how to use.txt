Manual Windows setup

Use this only if the one-click script does not work on your computer.

1. Open a terminal in the project folder.

Example:
~\Python-Exam-Agent

2. Create a virtual environment.

Command:
python -m venv .venv

3. Activate the virtual environment.

Command:
.venv\Scripts\activate

4. Install dependencies.

Command:
pip install -r requirements.txt

5. Start the backend server.

Command:
cd backend
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

6. Open a second terminal in the project folder.

Example:
~\Python-Exam-Agent

7. Start the frontend server.

Command:
cd frontend
python -m http.server 3000

Exam flow

1. Click Start Exam.
2. Click Start 8-Question Exam.
3. Complete the exam within 90 minutes.
4. Use Previous and Next to move one question at a time.
5. Read the visible examples before writing code.
6. Write code in the editor.
7. Click Compile & Run Tests.
8. Review visible test results and hidden backend edge test status.
9. Click Finish Exam to see the final score.

AI question generation

1. Click AI Generator.
2. Enter your API key.
3. Choose a model version.
4. Click Generate AI Questions.
5. The generated questions are drafts based on the existing built-in question bank.
6. The API key is used for that request only and is not saved in the database.

Generate full exam example

{
  "topic": "__all__",
  "difficulty": "mixed",
  "number_of_questions": 8,
  "username": "test"
}

Notes:
- The default test user is created automatically when the backend starts.
- The built-in question bank is stored in backend/app/services/question_generator.py.
- A new device can clone the GitHub repository, start the backend, and use the same question bank.
- Each exam draws 8 questions from the built-in question bank.
- Difficulty is stored for every question in the database.
- Each exam contains 3 easy questions, 3 medium questions, and 2 hard questions.
- At most 2 questions can share the same type/topic.
- There is no topic or question-count selection in the frontend exam.
- Use "mixed" as the difficulty value.
- Use "__all__" as the topic value to start the full 8-question exam.
- The frontend exam timer is 90 minutes.
- The final score is shown as 1/8, 2/8, and so on. A full score shows PASS.
- Some backend edge test cases are hidden, so students cannot see every test input.
- Current topics include lists, integers, number theory, file processing, matrices, patterns, and sets and words.
- Some questions use temporary files such as CSV files or dictionary.txt during automatic grading.
- The local SQLite database is created automatically when the backend starts.
